WHY ASTRO OVER NEXT.JS
Next.js is great but overkill here. The site is 90% static content â€” Astro renders it all at build time with no client JS, giving perfect Lighthouse scores out of the box. When you need interactivity (booking embed, course player, admin panel), Astro's island architecture lets you hydrate just those components. You also avoid the complexity of Next.js server components, caching quirks, and heavier deployment requirements. KEY DESIGN DECISIONS
Content lives in Markdown
Blog posts are .md files in src/content/blog/. Brooke can write in any text editor, or you can set up a simple headless CMS like Tina or Decap (both free, Git-based) that gives her a visual editor. No database needed for content.
Interactive parts are React islands
Only the booking widget, contact form, and course player ship JavaScript to the browser. Everything else is pure HTML/CSS. This means the site loads fast even on slow connections â€” great for mobile users Googling "guitar lessons near me."
